# Technical Deep Dive: Choice C + B Architecture

OpenIssue AI has transitioned from a heuristic keyword-matching system to a **Hybrid Semantic & LLM Reasoning Engine**. This document provides a detailed breakdown of the technical components and their interactions.

---

## 🌀 1. AI Engine: Choice C (Semantic Vector Similarity)
Instead of matching strings (e.g., "crash", "error"), we now use high-dimensional mathematics to understand the **Intent** of an issue.

### How it works:
1.  **Vectorization**: Every incoming issue (Title + Body) is converted into a **384-dimensional floating-point vector** using the `all-MiniLM-L6-v2` Sentence Transformer.
2.  **Category Seeds**: We've defined a set of "Seed" strings for each category (`Bug`, `Feature`, `Question`, `Query`, `Procedure`, `Method`).
3.  **Cosine Similarity**: The system calculates the inverse angular distance (Cosine Similarity) between the issue's vector and the category's seed vectors.
4.  **Mathematical Winner**: The category with the highest average similarity score is chosen.

---

## 🧠 2. AI Engine: Choice B (LLM Structured Reasoning)
While Choice C understands the *Category*, Choice B (Gemini 1.5 Flash) provides the **Deep Reasoning**.

### Technical Details:
- **JSON Output Mode**: We configured the Gemini API with `response_mime_type: "application/json"`. This ensures the AI always returns a machine-readable object, not just a chat response.
- **Structured Logic**: The AI is prompted to return a specific schema:
  - `classification`: Confirms or overrides the Choice C label.
  - `priority`: Analyzes severity (Low, Medium, High, Critical).
  - `tech_domain`: Identifies the stack (e.g., `Rust`, `React`, `Python`).
  - `response`: Generates the professional, technical maintainer reply in Markdown.

---

## 🌐 3. Autonomous Webhook & Security
The system is now production-grade and secure against external attacks.

### Components:
- **HMAC-SHA256 Signature**: Every webhook request from GitHub is signed with your `GITHUB_WEBHOOK_SECRET`. Our backend computes the signature locally and compares it to the incoming header.
- **Async Pipeline**: The webhook endpoint is non-blocking, triggering the `IssueAnalyzer` asynchronously to avoid timeouts.

---

## 🕵️ 4. Web Suggestions & Domain Locking
This feature eliminates irrelevant "Web Suggestions" (false positives).

### The Logic:
1.  **Domain Identification**: The LLM (Choice B) identifies the **`tech_domain`** (e.g., `Rust-Programming`).
2.  **Locking**: The backend looks at its library of `tech_patterns`. If a pattern matches the identified domain, it is **locked**.
3.  **Regex Hardening**: Fallback keyword matching now uses **Word Boundary Regex** (`\b`).
